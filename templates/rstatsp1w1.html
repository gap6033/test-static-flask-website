Population Data v. Sample Data

Sample data must be representative of Population; no sort of sampling bias or any kind of exclusions, to be as random as possible.

The data points in a sample must be independent of each other.

Samples always give an estimation/approximation of the population data rather than the exact representation. There always exist some sort of unknown uncertainties in samples. Due to this reasons formulas involving population data and sample data have slight differences.

Categorical Data – Labels, Names or other descriptors. There is no order(unlike Quantitative Data) involved and there exists exclusivity in theses description; no overlapping.

Measures of Variability: Z-Scores

Standard Deviation/Variance tells about how spread/distributed out the data is. Two data sets can have the same mean, but their spread may differ. This spread is also referred to as Variability (one of the measures of variability). A low Standard Deviation means the data is clustered towards the mean and a High Standard Deviation means the data is spread apart from the mean.

Z-score tells about how much spread out a single data point is from the mean. The no. of standard deviations away is a data point from the mean. It is a unit less measure (unlike standard deviation) and hence, allows us to compare data sets that may even be in different units.

 

Coefficient of Variation – S.D. * 100 /Mean, How large is the Standard Deviation relative to the Mean. It is also unit independent; what units our original data is when comparing data sets.

Random Variable Basics

A variable that takes on a numerical value as a result of random experiment; there is a numerical value associated with each possible outcome. Presence of NUMERICAL VALUE is ESSENTIAL.

2 Concepts to be kept differentiated –

·         The numerical value that the random variable can take associated with each possible outcome. Eg – values 1 to 6 when throwing a dice or Heads or Tails in a Coin Flip.

·         The Random Variable itself. Eg – the act of throwing the dice itself.

Discrete Random Variable –

Imagine rolling a dice; we only have 6 possible outcomes. So we get basically get a cube with six sides.

Variables that have finite no. of outcomes with no in-between or variables with a series of infinite outcomes but only whole no.s. The magnitude of difference b/w the outcome values is meaningful.

Continuous Random Variable –

Imagine an experiment (eg - temperature outside) where there are infinite possible outcomes (imagine a dice where there are infinitely many surfaces and the difference between the values of each of the surface is very very minute, that ultimately turns the cube dice into a smooth sphere), basically between successive possible outcomes, the magnitude of difference is miniscule. Here, there exist in-betweens and also values between whole no.s..

Combinations – Under the Normal Curve

 

Basically tells how the area under the Normal Curve comes to be 1. It converts all the possible combinations into proportions and then plot a histogram. Thereafter, it equates the Histogram to a Normal Curve (something we look for when calculating probabilities). Sum of all possible Proportions and Probabilities in a data is equal to 1. Hence, the area under the Histogram or a Normal Curve is equal to 1.

One is histogram version of the proportions, one is a representation with a smooth line, and both have area under them equal to 1 (as we are visualizing proportion).

This area = 1, represents probabilities/proportions.

R basic knowledge

  • vectors - assigning same type of values to an object eg - x <- c(7, 2, 3, 4)
  • executing it (pressing ctrl+enter), the environment will look like x |num[1:4} 7 2 3 4 and the console will simply display x <- c(7, 2, 3, 4) as it is. However, calling out x now in the script will print values 7 2 3 4 in the console.
  • We can use index to call specific values for eg x[1] will give 7 in the console or x[1:4] will give 7 2 3 4.
  • Using operators (<, >, ==, !=) will give boolean outputs in the console for each of the values, eg, x > 0 will give True True True True.
  • x^2 will square each of the values, same goes for multiplication, division, subtraction, addition.
  • Mean(x) to get mean of all the values.
  • length(x) will give the total number of items. sum(x) will add all the items.
  • We can combine indexing and other functions to get desired outputs, eg - x[x>0], rather than boolean output this will give all the numbers inside the vector that are greater than zero.
  • Importing data, the file should be in .csv, you can save excel file as .csv. Import Dataset option in the environment will allow to import data. Once imported a window pops up, asking for name, whatever name you give will then become the object inside the Rstudio. Selecting Headings as yes will ensure that the 1st row of the dataset contain column names (aka the variables). The environment will then tell you the number of rows (obs for observation) and number of columns(the variable names).
  • Using a function on a dataset involves - functionname(datasetname$variablename), eg - mean(Bikedata$distance), this would be an example representing a quantitative variable.
  • Representing a categorical variable can be done using table function, eg - table(datasetname$variablename). Executing it would display in the console each of the sub-categories within that categorical variable alongwith the total count of each of those sub-categories.
  • INDEXING DATA FRAMES - data frame is a matrix of multiple rows and columns. Pulling out specific cases from a data set also involves indexing - dataframename[row no., column name]- the first index would always refer to the row and the 2nd to column.
  • Pulling out the entire data of one single variable(column) - dataframename[, column name] - notice, the first index is kept blank. This will represent the data as it is in that column but not the total count of each of the values.
  • To get a total count of cases inside a variable, we need to use the table function again - table(datasetname$variablename)
  • CREATING NEW DATAFRAME from existing one -
    We can combine assignment and indexing in dataframes to pull out data pertaining to specific cases, eg- data pertaining to females. females <- Bikedata[Bikedata$gender == 'F', ] - keeping the 2nd index blank will allow us to include all the other columns. Running it would create a new dataset in the "environment" with the dataset name females. Basic format, datasetname you want <- existing datasetname[specific row item you want inside a variable, the column(s) you want for that row item], now this would create an entire new dataset with all the other variables.
  • If you want a data set for only one other variable, instead of datasetname, you can make the format as dataname <- [datasetname$variable, datasetname$diffvariable == 'F'], here we are only categorizing females based on speed and hence, there is no second index to refer to some other column. Once we have narrowed down the dataset to a single column we can perform functions such as mean(assignmentname).
